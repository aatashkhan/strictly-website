## Project Overview

**Live site:** Deployed on Vercel
**Repo:** https://github.com/aatashkhan/strictly-website

- **AI:** Anthropic Claude Sonnet 4 via `@anthropic-ai/sdk`
- **Maps:** Mapbox (itinerary maps), Google Maps API (Places autocomplete for hotel search)
- **Auth:** Supabase Auth with Google OAuth
- **Deployment:** Vercel

## Architecture Decisions

### Admin Auth Flow
`adminAuth.ts` extracts the Supabase access token from auth cookies, verifies the user's identity using the anon key, checks `admin_users` table, then returns a **service-role Supabase client** that bypasses RLS for all admin data operations.

### Supabase Typing
No generated Supabase types — admin API routes use `/* eslint-disable @typescript-eslint/no-explicit-any */` and `createClient<any>()`. This is acceptable for internal admin routes.

### AI Voice Settings
`buildSystemPrompt()` in `prompts.ts` accepts optional `VoiceSettings` loaded from the `site_content` table (section: `ai_voice`). Both `/api/generate` and `/api/chat` routes load these settings before building prompts. Denna can change the AI's personality, vocabulary, tone via admin without a deploy.

### Route Optimization
`routing.ts` uses meal-period-aware nearest-neighbor: groups itinerary items into 5 time buckets (morning/midday/afternoon/evening/night), optimizes within each bucket by geographic proximity, then reassembles in chronological order.

## Current State

### What's Working
- Full trip planning flow (form → AI generation → itinerary display → chat refinement)
- Trip saving, sharing, "Today" view
- Admin panel: venue CRUD, filters, sorts, review mode, AI chat assistant
- Admin site content CMS with auto-save and AI voice preview
- Hotel picker with expandable cards, city/getaway split
- City-contextual vibes (custom + auto-generated from category counts)
- Multi-select transit preferences
- Database-driven AI voice settings

### What's Partially Done
- **Site components don't read from `site_content` yet** — Hero, Footer, About, ThreeCards still use hardcoded text. The data is seeded in the DB and editable in admin, but components need to be converted to server components or receive data as props to actually use it. The AI voice settings DO work live.
- **Featured cities drag-and-drop** (spec 10G) — Not implemented. Currently uses `FEATURED_CITIES` constant.

### Known Issues
- `<img>` warnings in VenueEditor and HotelPicker (using `<img>` for dynamic URLs instead of `next/image` — acceptable)
- `saveTimeoutRef.current` warning in trips page (pre-existing, cosmetic)
- NYC has only 24 venues (all eat/drink, no stay/explore/shop/spa)
- Ojai has only 18 venues (no drink/explore/spa)

### Admin Access
Current admins are set via the `admin_users` table in Supabase. To add an admin:
1. User signs in at `/admin` with Google
2. Find their UUID in Supabase → Authentication → Users
3. `INSERT INTO admin_users (id, email, role) VALUES ('uuid', 'email', 'admin');`
